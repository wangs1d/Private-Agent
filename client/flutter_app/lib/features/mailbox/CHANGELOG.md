# 邮箱Tab更新说明 - 新朋友功能优化

## 更新日期
2026年5月16日

## 更新内容

### 主要变更
将原来的"收到的请求"和"发出的请求"两个Tab合并为一个**"新朋友"**Tab，统一展示所有好友请求。

### 改进点

#### 1. 更简洁的界面
- **之前**: 3个Tab（好友 / 收到的请求 / 发出的请求）
- **现在**: 2个Tab（好友 / 新朋友）

#### 2. 统一的请求管理
在"新朋友"Tab中，你可以看到：
- ✅ **收到的请求** - 别人发给你的好友请求
  - 显示"收到"标签（蓝色）
  - 可以接受或拒绝
  
- ✅ **发出的请求** - 你发给别人的好友请求
  - 显示"发出"标签（绿色）
  - 显示当前状态（等待回应/已接受/已拒绝/已取消）

#### 3. 智能区分
每个请求卡片都会清晰标识：
- **方向标识**: "收到"或"发出"标签
- **不同颜色**: 
  - 收到的请求使用蓝色系
  - 发出的请求使用绿色系
- **操作按钮**: 只有收到的待处理请求才显示"接受/拒绝"按钮
- **状态标签**: 发出的请求和非pending状态的请求显示状态标签

### 技术实现

#### 代码变更
1. **数据结构简化**
   ```dart
   // 之前
   List<Map<String, dynamic>> _incomingRequests = [];
   List<Map<String, dynamic>> _outgoingRequests = [];
   
   // 现在
   List<Map<String, dynamic>> _allRequests = [];
   ```

2. **API调用优化**
   ```dart
   // 之前：分别调用两个API
   await widget.api.getIncomingFriendRequests();
   await widget.api.getOutgoingFriendRequests();
   
   // 现在：一次调用获取全部
   await widget.api.getAllFriendRequests();
   ```

3. **智能渲染逻辑**
   ```dart
   // 自动判断是收到还是发出的请求
   final bool isIncoming = fromActorId != ApiConfig.effectiveActorId;
   
   // 根据类型显示不同的UI
   if (isIncoming && status == "pending") {
     // 显示接受/拒绝按钮
   } else {
     // 显示状态标签
   }
   ```
