# Agent能力说明优化 - 隐藏技术细节

## 🎯 优化目标

将**底层技术细节**从Agent向用户展示的能力清单中移除，只保留**面向用户的功能描述**。

---

## ✅ 修改内容

### 1. AIP协议 → 多Agent协作能力

**修改前（技术术语）：**
```
5️⃣ 【AIP协议能力】
可用工具：
- aip.dispatch: 分发任务到其他Agent
- aip.list_my_state: 查看当前Agent状态
- aip.get_proposal: 获取协作提案
提示：用于多Agent协作场景
```

**修改后（用户友好）：**
```
5️⃣ 【多Agent协作能力】
可用功能：
- 与其他Agent协同完成任务
- 组建团队共同解决问题
提示：当任务需要多个Agent配合时自动使用
```

**改进点：**
- ❌ 移除了 "AIP协议" 这个技术术语
- ❌ 移除了具体的工具名称（aip.dispatch等）
- ✅ 保留了核心功能描述
- ✅ 用用户能理解的语言表达

---

### 2. 协议统一管理 → 系统管理能力

**修改前（技术术语）：**
```
🔟 【协议统一管理能力】
可用工具：
- protocol.unified.quota_adjust: 调整计算配额
- protocol.unified.memory_patch: 更新记忆
- protocol.unified.memory_get: 获取记忆
- protocol.unified.human_directive: 接收人类指令
- protocol.unified.governance_probe: 治理探测
提示：用于系统级管理和配置
```

**修改后（用户友好）：**
```
🔟 【系统管理能力】
可用功能：
- 管理记忆和上下文
- 调整系统配置
提示：用于优化Agent性能和个性化设置
```

**改进点：**
- ❌ 移除了 "协议统一管理" 这个抽象概念
- ❌ 移除了所有protocol.unified.*工具名称
- ❌ 移除了 "治理探测"、"人类指令" 等技术术语
- ✅ 简化为用户能理解的"记忆管理"和"系统配置"
- ✅ 强调了对用户的价值（优化性能、个性化）

## 💡 设计原则

### ✅ 应该展示的
- 功能能做什么（What）
- 对用户有什么好处（Why）
- 什么时候会使用（When）

### ❌ 不应该展示的
- 底层技术实现（How）
- 具体的工具/函数名称
- 协议名称、技术术语
- 内部架构细节

## 📝 维护建议

当添加新能力时：

### ✅ 推荐写法
```typescript
lines.push(`\n🆕 【XXX能力】`);
lines.push(`可用功能：`);
lines.push(`- 功能描述1（用户能理解的）`);
lines.push(`- 功能描述2（强调价值）`);
lines.push(`提示：使用场景说明`);
```

### ❌ 避免写法
```typescript
lines.push(`\n🆕 【XXX协议/框架】`);
lines.push(`可用工具：`);
lines.push(`- xxx.tool_name: 技术描述`);
lines.push(`- yyy.api_call: 内部实现`);
```
