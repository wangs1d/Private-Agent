# 子 Agent 场景化分类重构说明

## 🎯 重构原因

原有的 14 种子 Agent 过于细分，导致：
- ❌ 任务分解复杂度高
- ❌ 路由选择困难
- ❌ 用户理解成本高
- ❌ 维护难度大

## ✅ 新架构：8大生活场景分类

按照用户的**实际生活场景**进行分类，更符合人类思维习惯：

### 📋 8种子 Agent 类型

| 类型 | 名称 | 覆盖范围 | 典型任务 |
|------|------|---------|---------|
| **life** | 生活助手 | 个人日常事务 | 天气查询、日程安排、提醒设置、闹钟、健康管理 |
| **work** | 工作助手 | 办公职业相关 | 文档处理、邮件管理、会议安排、报告生成 |
| **social** | 社交助手 | 人际互动 | 消息发送、朋友圈动态、联系人管理 |
| **entertainment** | 娱乐助手 | 休闲娱乐 | 游戏、音乐、视频、休闲活动 |
| **finance** | 金融助手 | 财务相关 | 钱包管理、支付转账、交易投资、购买技能 |
| **tech** | 技术助手 | 技术开发 | 代码生成、桌面控制、截图、视觉识别 |
| **info** | 信息助手 | 信息查询 | 网络搜索、翻译、知识问答、新闻资料 |
| **general** | 通用助手 | 其他任务 | 未分类的通用对话 |

---

## 🔄 对比示例

### 旧架构（14种）
```
用户："明天北京天气怎么样？帮我设个早上8点的闹钟"

分解：
├─ [weather] 查询天气
└─ [calendar] 设置闹钟
```

### 新架构（8种）
```
用户："明天北京天气怎么样？帮我设个早上8点的闹钟"

分解：
└─ [life] 生活助手处理（天气+闹钟都属于生活事务）
```

**优势：** 更简洁，符合用户直觉！

└─ [life] 生活助手

```

主 Agent 分解：
└─ [entertainment] 娱乐助手
   ├─ 启动斗地主游戏
   └─ 播放音乐
```

## 🎨 设计理念

### 1. 以用户为中心
按**生活场景**而非**技术功能**分类，更符合用户思维：
- 用户不会想"我需要 weather agent"
- 用户会想"我需要处理生活事务"

### 2. 减少认知负担
从 14 种降到 8 种，降低：
- 主 Agent 的决策复杂度
- 用户的理解难度
- 系统的维护成本

### 3. 保持专业性
每个场景助手内部仍然专业化：
```typescript
// life 助手内部整合了多个专业工具
tools: [
  ...calendar tools,    // 日程管理
  ...weather tools,     // 天气查询
  ...reminder tools,    // 提醒设置
]
```

### 4. 易于扩展
未来可以添加更多场景：
- `health` - 健康助手
- `education` - 教育助手
- `travel` - 旅行助手
- 等等...

## 📊 性能影响

### 优势
✅ **更快的路由决策** - 8选1比14选1更快  
✅ **更高的命中率** - 场景覆盖更广，减少误判  
✅ **更好的用户体验** - 符合直觉，易于理解  
✅ **更低的维护成本** - 类型更少，更易管理  

### 无明显劣势
- 每个场景助手内部仍然调用专业工具
- 专业性不受影响
- 执行效率保持一致

## 🔧 技术实现

### 类型定义
```typescript
export type SubAgentType = 
  | "life"          // 生活助手
  | "work"          // 工作助手
  | "social"        // 社交助手
  | "entertainment" // 娱乐助手
  | "finance"       // 金融助手
  | "tech"          // 技术助手
  | "info"          // 信息助手
  | "general";      // 通用助手
```

### 能力配置
每个场景助手整合相关工具：
```typescript
capabilities.set("life", {
  type: "life",
  name: "生活助手",
  description: "处理个人生活事务：天气、日程、提醒等",
  keywords: ["天气", "日程", "提醒", "闹钟", "约会", "健身"],
  tools: [
    ...calendarTools,
    ...weatherTools,
    ...reminderTools,
  ],
});
```

### 智能路由
主 Agent 根据关键词匹配场景：
```
"天气" + "提醒" → life (生活助手)
"文档" + "邮件" → work (工作助手)
"游戏" + "音乐" → entertainment (娱乐助手)
```

## 📝 迁移指南

### 对于开发者
无需修改现有代码，只需更新提示词中的类型定义。

### 对于用户
完全透明，用户体验更流畅自然。

### 对于监控
指标统计仍然有效，只是 Agent 类型名称变化：
- 旧：`calendar`, `weather`, `wallet`...
- 新：`life`, `work`, `finance`...

---

## 🎯 总结

**从 14 种技术导向 → 8 种场景导向**

这是一个**以用户为中心**的重构，让系统：
- ✨ 更智能（符合人类思维）
- ✨ 更高效（减少决策复杂度）
- ✨ 更易用（降低理解门槛）
- ✨ 更易维护（简化架构）

**核心思想：** 不要让用户适应系统，要让系统适应用户！
