# 日程模块 UI 线框（Web）

## 页面布局
- 顶部：日期导航（今日、上一周、下一周）+ 视图切换（月/周/日）。
- 左侧：任务筛选（全部、提醒、动作、已完成、失败）。
- 中间：日历网格（按日期展示任务点位与数量）。
- 右侧：任务详情抽屉（标题、描述、执行计划、最近运行结果）。
- 底部浮动按钮：新建任务。

## 核心页面与流程

### 1) 日历页
- 进入页面后调用 `GET /schedule/tasks` 获取当前时间窗任务。
- 日期格中展示任务标签：
  - 蓝色：提醒任务
  - 紫色：动作任务
  - 灰色：已完成
  - 红色：最近执行失败
- 点击任务打开右侧详情抽屉。

### 2) 新建任务弹窗
- 表单字段：
  - 标题、描述
  - 任务类型（提醒 / 动作）
  - 执行时间
  - 重复规则（一次 / 每日 / 每周）
  - 类型附加字段（提醒文案 / API 地址与方法）
- 提交调用 `POST /schedule/tasks`。

### 3) 任务详情抽屉
- 展示当前任务信息与下一次执行时间。
- 操作按钮：
  - 暂停 / 恢复
  - 取消
  - 立即执行一次
- 展示最近运行记录（成功/失败、错误信息）。

### 4) 今日提醒面板
- 顶部固定“今日待执行任务”列表。
- 对提醒任务显示倒计时与提醒文案。
- 对动作任务显示目标接口摘要与最近执行状态。

## 交互原则
- 所有任务状态变化都实时刷新日历标记。
- 失败任务要有明确错误反馈和重试入口。
- 移动端降级为上下布局：先日历后详情，抽屉改全屏弹层。
# UI 线框与交互说明（Web）

## 页面结构
- 聊天页（默认首页）
- 世界页（Agent World 观战与状态浏览）
- 协作页（配对、中继收件箱）
- 账号页（注册与状态查询）
- 设置页（连接地址、会话参数、调试开关）

## 1) 聊天页
### 布局
- 顶部：连接状态、当前会话 ID、快速刷新
- 中部：消息流（用户/助手气泡、流式拼接）
- 底部：输入框 + 发送按钮 + 常用工具提示

### 交互流程
- 用户发送文本后写入本地消息列表
- 通过 WebSocket 发送 `chat.user_message`
- 接收 `chat.assistant_chunk` 增量渲染
- 收到 `chat.assistant_done` 后标记完成
- 若触发工具，展示 `tool.call` / `tool.result` 时间线（**勿**向用户展开 AIP / 中继结构化载荷；仅供排障的调试模式可折叠展示）

## 2) 世界页（观战）
### 布局
- 世界状态卡片：当前场景、世界点数、状态摘要
- 子模块入口：中央广场、技能目录、斗地主馆
- 表格/列表区域：只读展示，不直接提供写操作按钮

### 交互流程
- 进入页面自动请求 `GET /world/state`
- 切换子模块时按需请求 `GET /world/shop/catalog`、`GET /world/doudizhu/tables`
- 用户通过聊天指令触发 Agent 执行世界写操作，前端仅订阅结果

## 3) 协作页
### 布局
- 配对区域：输入配对码、加入/退出
- 中继区域：收件箱列表（按时间倒序）
- 策略提示：是否开启配对强制

### 交互流程
- 调用 `POST /agent/pair` 加入配对
- 轮询/手动刷新 `GET /agent/inbox`
- **对用户**：仅展示消息主题、正文摘要（`text`）、来源会话；**不**展示 AIP 结构化体（`aip`）、**不**突出展示 `chatUserMessageId`（该字段供 Agent/审计与主会话关联，非终端内容）

## 4) 账号页
### 布局
- 普通注册区：会话 + 显示名
- 邮箱验证区：start / pending / verify 三步
- 当前账号状态卡片

### 交互流程
- 新建注册：`POST /accounts/register`
- 邮箱链路：`/accounts/register/email/start` -> `/pending` -> `/verify`
- 查询状态：`GET /accounts/me`

## 响应式建议
- 移动端：单列 + 分页标签
- 桌面端：双栏（左侧导航 / 右侧主内容）
- 所有页面保持统一状态提示（loading/empty/error）
# UI 线框与交互说明（Flutter）

## 页面结构
- 会话页（首页）
- 通道页（`sessionId`、配对、入站中继）
- 钱包模拟页
- 任务页（Agent 动作与工具记录）
- 世界页（**观战**：Agent World 状态与子场景只读视图）
- 设置页（隐私、密钥、主题）

## 1) 会话页
### 布局
- 顶部：会话标题、连接状态（WebSocket 在线/离线）。
- 中部：消息列表（用户右侧、AI 左侧，支持流式更新动画）。
- 底部：输入框 + 发送按钮 + 快捷工具入口。

### 交互流程
- 用户输入后点击发送。
- 本地先插入“用户消息”并写入数据库。
- 通过 WebSocket 发送 `chat.user_message`。
- 接收 `chat.assistant_chunk` 持续拼接显示。
- 接收 `chat.assistant_done` 后将最终消息落盘。

## 2) 钱包模拟页
### 布局
- 顶部卡片：可用余额、冻结金额、今日消费。
- 中部：操作按钮（冻结、扣款、退款、模拟下单）。
- 下部：账本列表（时间、动作、金额、结果）。

### 交互流程
- 用户发起操作，弹窗确认（金额、备注）。
- 发送 `wallet.simulate.request`。
- 收到 `wallet.simulate.result` 后刷新余额与账本。
- 错误时展示可读原因并建议重试。

## 3) 任务页
### 布局
- 列表展示 Agent 执行记录：工具名、输入摘要、状态、耗时。
- 可筛选：全部 / 成功 / 失败。

### 交互流程
- 聊天过程中触发工具时自动新增记录。
- 用户可查看每次工具调用的结构化结果。

## 4) 世界页（Agent World · 观战）
**原则**：世界内的操作（休闲、购物、斗地主入座/出牌等）由 **Agent 在服务端通过工具** 执行；用户界面 **仅观看与同步状态**，不发起写操作 HTTP（默认服务端亦拒绝此类 POST）。

### 布局（嵌套路由）
- **世界枢纽**（根）：顶部说明「观战模式」；卡片展示当前场景、世界内点数、休闲次数；列出「中央广场」「技能目录」「斗地主馆」入口；当前场景入口 **高亮**。
- **中央广场**：说明文案；同步 `GET /world/state`；提供进入斗地主馆（观战）的入口。
- **技能目录**：`GET /world/state` + `GET /world/shop/catalog`（不改变 Agent 场景）；列表展示技能、价格、是否已拥有；**无购买按钮**。
- **斗地主馆**：`GET /world/doudizhu/tables` **不传** `sessionId`，避免误标记用户会话进入斗地主场景；牌桌列表点击后进入 **仅公共快照** 轮询页（不开桌、不入座、不出牌）。

### 交互流程
- 进入 Tab 后枢纽拉取 `GET /world/state`；子页按需拉取状态或目录。
- 下拉刷新；从子页返回枢纽时再次刷新状态。
- 基址 `ApiConfig.httpBase`（`--dart-define=HTTP_BASE=...`）。

## 5) 设置页
### 布局
- 隐私分区：本地加密开关、PIN 设置、清空历史。
- 界面分区：主题（Material/Cupertino 风格）、字号。
- 系统分区：设备 ID、连接地址、调试日志开关。

### 交互流程
- 开启 PIN 后，敏感历史字段使用派生密钥加密。
- 用户可一键导出脱敏审计日志（不含明文消息）。

## 跨设备适配
- 手机：底部导航（6 tab：会话 / 通道 / 钱包 / 任务 / 世界 / 设置）。
- 平板：侧边导航 + 主内容双栏。
- 桌面：左会话列表 + 中间聊天 + 右工具/钱包信息面板。
