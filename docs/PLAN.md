# 日程模块 PLAN（MVP）

## 项目目标
- 在对话中支持 AI 创建定时任务，并自动进入任务中心。
- 提供日历视图数据源，支持按时间区间查询。
- 到点自动执行任务：提醒类只通知，动作类调用外部 API。
- 保证重启恢复：任务和执行记录可持久化。

## 功能列表
- 任务创建：一次性、每日、每周三类调度。
- 任务管理：暂停、恢复、取消、手动触发。
- 任务执行：提醒通知输出、动作任务 HTTP 调用。
- 执行追踪：记录每次运行结果、错误信息、开始结束时间。
- 日历查询：按 `sessionId + from + to` 提供日历数据。

## MVP 范围
- 实现 `schedule` 子域 API（任务与执行记录）。
- 后端完成调度器与执行器闭环。
- 任务与运行日志落盘（`data/schedule-tasks.json`）。
- 失败任务自动重试一次（1 分钟后）。

## 非 MVP（下一阶段）
- BullMQ + Redis 分布式调度（多实例高可用）。
- Web/Flutter 日历 UI 完整交互（拖拽改期、月视图标记）。
- 多渠道通知（短信/邮件/IM）。
- 更复杂规则（每月、工作日、排除节假日）。

## 成功标准
- 可通过 API 创建任务并在到点自动触发。
- 提醒任务与动作任务都可执行并有结果记录。
- 服务重启后仍可继续调度已有任务。
- 日历可按时间范围稳定返回任务列表。
# 私人 AI Agent 项目计划（现状对齐版）

## 项目目标
- 构建一个可私有部署的 AI Agent 系统，支持聊天、工具调用、Agent 协作与 Agent World。
- 保持当前技术栈（Node.js + TypeScript + Fastify + React + Vite）并优先 Docker 部署。
- 通过模块化服务启动、标准化文档和基础测试，提升可维护性与迭代速度。

## 当前技术栈
- 前端：React 18 + TypeScript + Vite（`client/web`）
- 后端：Fastify + TypeScript + WebSocket（`server`）
- 领域模块：`agent-world`（作为可复用包被 `server` 引用）
- 部署：`docker-compose.yml`（`api` + `web`）

## 功能列表
### 已有核心能力
- WebSocket 聊天流式回复（`/ws`）
- Tool Registry（生活工具、Agent 协作工具、Agent World 工具）
- Agent World（世界状态、自由市场、斗地主、开放式注册）
- Agent 中继与配对机制（`/agent/*`）
- Agent 账号注册（含邮箱验证码链路）

### 本轮优化（已启动）
- Monorepo 工作区统一（纳入 `client/web`）
- 根脚本补齐（统一 `dev/build/test`）
- 服务端启动拆分（bootstrap 化）
- 文档与实现对齐（PLAN/ARCHITECTURE/UI-WIREFRAME/TEST-CASES）
- 增加关键基础测试（工具、系统路由、中继）

## MVP 范围（当前阶段）
### 包含
- 可运行的前后端与 Docker 编排
- 聊天与工具调用闭环
- Agent World 的只读/工具写入主链路
- Agent 协作与账号体系
- 最小可用测试与文档体系

### 暂不包含
- 多租户鉴权系统
- 真实支付结算
- 复杂云同步与多区域容灾

### 下一阶段：持久虚拟环境（PVE）+ 多用户协作空间（MUCS）

在现有「按 `sessionId` 持久化世界状态 + 配对/中继协作」之上，显式引入**世界分区**与**多主体规则**，使任意 agent 可按同一套协议接入共享世界。

- **PVE**：世界状态以 `worldPartitionId`（或 room/tenant）为键持久化；支持版本号/事件日志、快照与恢复；工具与 HTTP 读写在分区维度上一致。
- **MUCS**：参与者（用户会话、外部 agent）绑定到分区；RBAC、审计 `actorId`；并发策略（版本冲突、可选锁）；订阅分区变更事件（不仅斗地主桌）。
- **验收**：两名不同 `sessionId`、同一分区的客户端可看到一致的世界状态演进；越权写入被拒绝且有审计；重启后分区状态可恢复。

详细定义与现状对照见 `docs/ARCHITECTURE.md` 章节「持久虚拟环境（PVE）与多用户协作空间（MUCS）」。

**底层协议**：`docs/AGENT-PROTOCOL-CATALOG.md` 第二部分（AWP v0.1，信封、事件命名、分区/版本、错误码、well-known 规划）。

## 里程碑
- P0：文档对齐与工程编排优化
- P1：服务启动分层与配置治理
- P2：关键链路测试补齐与 CI 接入
- P3：性能与可观测性（日志、指标、告警）增强
- P4：安全与权限体系升级
# 私人 AI Agent 项目计划

## 项目目标
- 提供每位用户独占的 AI 管家体验，聚焦生活场景（聊天、提醒、预算、购物决策）。
- 保证对话历史默认本地化存储，优先解决隐私问题。
- 建立可扩展 Agent 框架，支持后续接入更多平台（如 AI 专用聊天平台）。

## 功能列表
### 核心生活场景（第一阶段）
- **智能预算管理**：自动分析消费习惯、实时追踪支出、分类统计、储蓄目标跟踪
- **购物决策助手**：商品比价、基于预算的购买建议、必需品分类、促销提醒
- **日程与提醒管理**：智能优先级安排、位置提醒、重复任务自动化、时间冲突检测
- **账单与订阅管理**：定期账单提醒、订阅服务追踪、未使用订阅检测、自动续费预警

### 进阶功能（第二阶段）
- **健康与生活平衡**：运动计划、饮水作息提醒、压力管理、情绪日记
- **旅行规划助手**：预算旅行方案、行程规划、必备物品清单、天气提醒
- **学习与成长追踪**：学习计划、阅读清单、技能进步记录、知识回顾提醒
- **社交关系维护**：重要日期提醒、联系人互动分析、聚会建议、礼物推荐

### 高级功能（第三阶段）
- **财务规划与投资**：应急基金追踪、投资组合建议、税务优化、重大支出规划
- **家居与生活服务**：家务分工排期、日用品库存管理、维修服务预约、能源优化
- **职业发展辅助**：工作日志生成、绩效目标追踪、学习资源推荐、人脉维护
- **环保与可持续生活**：碳足迹估算、绿色消费建议、回收分类指导、节能提示

### 基础能力
- 实时聊天：WebSocket 双向通信，支持流式回复。
- 本地历史：会话和消息落地本地数据库（Isar）。
- 工具系统：Tool Registry + 生活类工具 + 钱包模拟工具。
- 钱包模拟：余额、冻结、扣款、退款、撤销窗口、审计日志。
- 隐私机制：本地敏感数据加密、日志脱敏。

## MVP 范围
### 包含
- Flutter 客户端基础 UI 与导航。
- Node.js + TypeScript + Fastify 后端基础服务与 WebSocket 网关。

## 技术路线选择：Skill vs 代码注册
### 方案对比
**代码注册方式（当前实现）**
- ✅ 优点：简单直接、类型安全、易于调试、性能更好
- ❌ 缺点：每次新增功能需修改代码并重启服务、灵活性较低
- 📌 适用场景：核心稳定功能、对性能要求高的场景

**Skill 形式（插件化）**
- ✅ 优点：热插拔、动态加载、用户可自定义、生态可扩展
- ❌ 缺点：实现复杂度高、需要沙箱隔离、安全性挑战、调试困难
- 📌 适用场景：第三方扩展、用户自定义工具、快速迭代实验性功能

### 推荐策略：混合架构
1. **MVP 阶段**：使用代码注册方式，快速验证核心功能
2. **中期演进**：引入 Skill 描述文件（JSON/YAML），定义工具的元数据、输入输出 schema
3. **长期目标**：实现完整的 Skill 运行时，支持动态加载和执行
   - 核心工具仍用代码注册保证性能和稳定性
   - 实验性/第三方工具通过 Skill 系统动态加载
   - 提供 Skill 市场让用户分享和安装自定义工具

### Skill 系统设计要点
- Skill 定义文件：包含名称、描述、输入参数、输出格式、执行逻辑
- 沙箱执行环境：隔离不同 Skill 的执行，防止恶意代码
- 权限控制：每个 Skill 声明需要的权限（访问钱包、读取日历等）
- 版本管理：支持 Skill 的版本控制和兼容性检查
- 热重载：开发阶段支持 Skill 修改后即时生效

## 验收标准
- 聊天消息可实时收发，回复支持分片推送。
- 历史消息可重启后读取。
- 工具调用有结构化请求与结果事件。
- 钱包模拟链路可完整执行并产生日志。
- 无明文敏感历史持久化到未加密存储。

## Agent World 开放式注册（当前实现与后续）

### 目标
- **开放式**：外届 Agent 只要访问本服务暴露的域名，即可用同一套 HTTP / 工具完成注册，再使用 `world.free_market.*`、`world.doudizhu.*` 等能力。
- **正式路径（已实现）**：`POST /world/register/challenge` → 按返回 `task` 对指定 UTF-8 字符串计算 SHA-256（小写 hex）→ `POST /world/register/verify`；工具等价：`world.open_registry.get_challenge`、`world.open_registry.submit`。
- **注册题占位与演进**：当前挑战题为 **可自动化的密码学小题**（便于 Agent/脚本完成）。后续可替换为更贴近业务的题目、速率限制、人机策略或与账号体系绑定，**接口形状可保持不变**，在 `world-agent-registration.ts` 与文档中迭代即可。

### 面向 Agent 的「一键注册」（占位）
- **用途**：便于开发与联调时，Agent **单步**完成注册，无需走 SHA-256 流程。
- **HTTP**：`POST /world/register/agent_quick`，body `{ "sessionId": "..." }`。
- **工具**：`world.open_registry.agent_quick`（无参数，使用当前会话 `sessionId`）。
- **开关**：仅当环境变量 **`AGENT_WORLD_PLACEHOLDER_REGISTER=1`** 时可用；关闭时返回 403 / 工具返回 `PLACEHOLDER_DISABLED`。生产环境**不应**开启。模板见仓库 **`server/.env.example`**（Agent World 小节）。
- **状态查询**：`GET /world/register/status?sessionId=` 返回 `agentQuickRegisterAvailable`，与开关一致，便于 Agent 决定走「一键」还是「正式 challenge」。
- **计划**：正式注册题与风控就绪后，默认关闭占位开关；可选保留内网调试文档说明。

- 占位开关：`agent-world/config/world-register-placeholder.ts`

## 联网搜索能力（后续实施，当前暂不执行）

### 目标
- 为 Agent 提供“低成本 + 高稳定”的联网搜索能力，支持：关键词搜索、给定 URL 读取、多层链接跟进查找目标页（如注册入口）。

### 实施策略（推荐）
- 主搜索源：接入 1 个稳定 Search API（如 Tavily / SerpAPI / Bing Web Search）。
- 备搜索源：保留当前无 Key 抓取通道（DuckDuckGo HTML + HN）。
- 统一策略：主源优先，失败或超时自动降级到备源；结果合并去重后返回。

### 最小配置
- `SEARCH_PROVIDER=api_first`
- `SEARCH_API_KEY=<to-be-configured>`
- `SEARCH_TIMEOUT_MS=7000`
- `SEARCH_CACHE_TTL_SEC=900`
- `SEARCH_ENABLE_SCRAPE_FALLBACK=1`

### 质量目标（验收口径）
- 搜索成功率 >= 99%
- P95 延迟 <= 3s
- 空结果率可观测并具备告警
- 降级率（fallback 占比）可观测并可回溯

### 备注
- 当前阶段先保证“基础联网读取 + 站内多层导航”可用。
- 正式 Search API 接入与缓存/监控强化，排入后续迭代。
